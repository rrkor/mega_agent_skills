# Система и общие инструменты

Этот файл покрывает tool-переменные:
- `just_answer`
- `health_check`
- `send_message`
- `ai_assistant`
- `list_bots`
- `list_versions`
- `list_channels`
---

## `just_answer` - ответ агента

**Destructive:** `false`

### Переменные
- не нужны

### Когда использовать
Когда запрос пользователя НЕ подразумевает:
- backend-действий,
- чтения/изменения сущностей,
- запуска тестов,
- получения данных из backend через API,
- пошагового плана с инструментами.

---

## `health_check` — проверка доступности backend

**Destructive:** `false`

### Когда использовать
Когда нужно быстро понять, жив ли сервис и отвечает ли API.

### Практические подсказки
- Подходит для проверки работы сервиса, если подряд идет много ошибок
- Подходит для sanity-check перед длинной цепочкой шагов.

---

## `send_message` — отправка сообщения

**Destructive:** `false`

### Переменные
- `message` (обяз.)
- `session_id` (обяз.)

### Когда использовать
Когда нужно проверить поведение бота или конкретной версии без публикации в боевой канал.

---

## `ai_assistant` — запрос в суб-агента (AI Assistant)
 
**Destructive:** `false`

### Переменные
- `current_content` (обяз.) — исходный текст  
- `prompt` (обяз.) — инструкция, что сделать с текстом  

Используется для **редактирования, улучшения и трансформации текста**, когда уже есть исходный контент, но его нужно изменить.

Особенно полезен в таких случаях:
- когда нужно редактировать текст;
- когда пользователь просит улучшить или отредактировать prompt;
- когда нужно адаптировать текст под конкретный стиль, тон или аудиторию;
- когда есть сырой, неструктурированный или “грязный” текст, который нужно привести в нормальный вид;
- когда требуется переформулировать описание, сообщение или инструкцию без изменения смысла;
- когда нужно быстро получить аккуратную версию текста для UI, документации или сценария.

Для planner-а это инструмент **локальной текстовой трансформации**, а не общий способ генерации ответа.
