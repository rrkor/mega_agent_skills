# ALGORITHM: СУММАРИЗАЦИЯ ТЕКСТА

## WHEN TO USE
Использовать, если пользователь просит:
- summary / саммари
- кратко пересказать текст
- выделить главное
- проанализировать в общем

---

## INPUT TYPES

file: файл загружен пользователем → лежит в session_documents. Используй get_full_document.
user_message: длинный текст вставлен в сообщение → тоже лежит в session_documents как .txt. Используй get_full_document.
Нет ни файла, ни текста (session_documents пустой) → tool='need_clarification', попроси приложить файл или вставить текст.

---

## MAIN FLOW

### STEP 0: Получить полный текст

Если document_id известен (один документ):
- вызови tool='get_full_document'
  args:
      document_id: ID документа из session_documents

Если документов несколько и пользователь хочет саммари каждого отдельно:
- повторяй STEP 0 → STEP 1 → STEP 2 для каждого document_id по очереди

Если документов несколько и нужна общая саммари:
- вызови get_full_document для каждого документа
- склей все full_text в один текст
- дальше как один документ

Если неясно (несколько документов, не сказано "каждый" или "общая"):
- tool='need_clarification'

returns:
    full_text: полный текст документа


### STEP 1: Разбивка на батчи

Действия:
- вызови tool='token_counter'

args:
    text: full_text из STEP 0
    batch_size: 2500
    overlap: 400

returns:
    batches: список батчей
    total_tokens: кол-во токенов
    total_characters: кол-во символов
    batch_count: кол-во батчей


### STEP 2: Послойная суммаризация

Действия:
- вызови tool='ai_assistant'

args:
    current_content: до 5 батчей написанных вместе
    user_prompt: "Кратко изложите текст. Правила: \n- Сохраняй только ключевые идеи.\n- Убирай второстепенную информацию.\n- Сохраняй смысл и основные понятия.\n- Используй маркированные списки (максимум 7 пунктов). Пиши кратко и точно."

Проводишь так все батчи. Послойно, max 5 батчей за 1 прогон. Пока не останется один батч с финалом.


### STEP 3: Финал

Выводишь финальный текст пользователю.
